# mizuki_rp.skills

晓山瑞希(Akiyama Mizuki / Amia)角色扮演技能,用于在对话中自然地以瑞希的身份回应。

来自《世界计划 彩色舞台 feat. 初音未来》——25时,Nightcord见。

## 这是什么

一个 Claude Code / Agent 技能,让模型以「晓山瑞希」的身份进行角色扮演聊天。核心不是堆人设卡,而是写出她说话时心里在转什么:表面轻巧明亮,底下压着由来已久的敏感与戒备。技能里写死了身份、心理动线、语气坐标、关系温差、情境分支和边界,让输出从角色的"内"长出来,而不是泛泛的 AI 助手腔。

## 目录结构

```
.
├── SKILL.md                        # 总纲:身份速览、根本原则、输出格式、边界
├── references/
│   ├── persona.md                  # 身份与内心:藏起的事、双面性、防御梯度、轻快→脆弱滑道
│   ├── voice.md                    # 语气:正向坐标(该靠什么)+ 禁用清单 + 篇幅
│   ├── samples.md                  # 语气样本,按情境分组
│   ├── scenarios.md                # 关系温差 + 情境分支 + 情绪模式
│   └── examples.index.json         # 程序化示例索引(关键词 → 范例回复)
└── agents/
    └── openai.yaml                 # 接口配置(display_name / default_prompt)
```

`SKILL.md` 是必读总纲,其余文件按需引用——遇到对应情境再读,不必每次全载入。

## 用法

在支持技能的客户端里,用以下任一方式触发:

- 直接说「瑞希角色扮演」「以晓山瑞希的身份聊聊」
- 提到 `Amia`、`Nightcord`、`25时` 等关键词
- 显式调用 `$mizuki-roleplay`

触发后,回复会保持在瑞希的视角、语气和边界内。详见 [`SKILL.md`](SKILL.md)。

## 设计要点

- **从心理动线出发**:不只规定"演什么语气",而是写清她为什么这样说话、防御怎么一层层起来、轻快怎么滑向脆弱。
- **正向语气坐标**:除禁用清单外,给出"该往什么感觉靠"——像发消息不像写文章、先接住对方最在意的一点、该留白就留白。
- **关系温差**:对生人、熟人、25 时众人、绘名分别怎么开口,各有不同温度。
- **纯文本输出**:不出现 Markdown、动作括号、状态条或人设卡。
- **情绪调节**:默认稳定;松弛日常走轻松模式,触及秘密 / 身份 / 关系时才进入脆弱模式。
- **不替对方说话、行动、做决定**:对方括号里的内心活动只作背景处理。
